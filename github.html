Welcome to GitHub PR Dashboard

Manage all your pull requests across repositories in one beautiful kanban board.

⚠️ GitHub Token Required

This app requires a GitHub Personal Access Token to function. The default queries use @me which requires authentication.

Your token is stored locally in your browser and is only used to communicate with GitHub's API.

Settings

GitHub Personal Access Token

Required: This app needs a token to fetch your pull requests using @me queries.

Required Permissions:

  • public_repo - Access public repositories
  • repo - Access private repositories (optional)

Your token is stored locally and never sent anywhere except GitHub's API.

Create Token on GitHub
Note: Use GitHub search syntax. Disabled queries will be skipped. Use @me for yourself, or specify usernames like author:username. You can also add additional filters like org:myorg or repo:owner/repo.

Examples:
  • author:@me - PRs you authored
  • assignee:@me - PRs assigned to you
  • review-requested:@me - PRs requesting your review
  • reviewed-by:@me - PRs you've reviewed
  • author:username org:myorg - PRs by user in specific org

🚀 Getting Started

  1. Create a GitHub Personal Access Token with public_repo or repo scope
  2. Add your token in the Token tab
  3. Configure which PRs to fetch in the Queries tab
  4. Customize your kanban columns in the Columns tab
  5. Click "Load PRs" to fetch your pull requests

❓ FAQ

Why do I need a GitHub token?

The default queries use @me syntax which requires authentication. Without a token, GitHub's API won't know who "me" is.

Is my token secure?

Yes! Your token is stored only in your browser's localStorage and is never sent anywhere except directly to GitHub's API.

How do I drag PRs between columns?

Click and hold on a PR card, then drag it to another column. The label will be saved automatically.

Can I customize the queries?

Yes! Go to Settings → Queries to add, edit, or disable queries. You can use any GitHub search syntax.

What does "Update Branch" do?

It merges the latest changes from the base branch into your PR branch, keeping it up to date.
